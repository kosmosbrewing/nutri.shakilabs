# SEO·신뢰 설계 — 11.nutri

- 상태: Draft v0.2
- 기준일: 2026-07-10

## 검색 전략

첫 클러스터는 영양제 종류 탐색과 “멀티비타민 추천”의 의료적 단정이 아닌 비교 의도를 충족한다.

| 의도 | 대표 주제 | 도착 페이지 |
|---|---|---|
| 탐색 | 영양제 종류, 비타민D·유산균·오메가3 제품 | `/categories`, `/categories/:slug` |
| 비교 | 멀티비타민 비교, 종합비타민 성분 비교 | `/nutri` |
| 가격 | 멀티비타민 1일 가격, 가성비 비교 | `/nutri` |
| 검증 | 멀티비타민 성분표 보는 법 | `/methodology` |
| 제품 | `{제품명} 성분 가격 비교` | `/products/:slug` |
| 신뢰 | 영양효율 점수 계산법 | `/methodology` |

## 색인 원칙

- 정적 HTML이 사용자 화면과 같은 본문을 제공해야 한다.
- 제품 상세는 완전성 gate 통과 후 sitemap에 추가한다.
- query 조합과 필터 결과는 별도 색인하지 않는다.
- 비교 페이지 canonical은 query 없는 `/nutri/compare`로 고정한다.
- 검색 수요만을 위한 얇은 파라미터 페이지를 만들지 않는다.
- 미존재 경로는 실제 404와 noindex를 반환한다.

## 메타데이터

- title은 핵심 비교 대상, 기준연도, 브랜드를 60자 안에서 설명한다.
- description은 1일 비용, 영양충족도, 공식 출처를 155자 안에서 설명한다.
- canonical은 실제 200 응답 호스트인 `https://shakilabs.com/nutri` 기준 self URL이다.
- OG URL과 canonical은 같아야 한다.
- `lastmod`는 실제 데이터나 본문 변경일만 갱신한다.

## 구조화 데이터

- 허브·랭킹: `WebApplication`, `ItemList`, `BreadcrumbList`
- 방법론·출처: `Article`, `BreadcrumbList`
- FAQ가 실제 표시될 때만 `FAQPage`
- 가격 신선도와 판매조건이 충분하지 않은 동안 `Product`·`Offer` schema는 사용하지 않는다.
- 자체 점수를 사용자 리뷰 평점처럼 `AggregateRating`으로 표시하지 않는다.

## 신뢰 UI

모든 랭킹 카드와 상세 페이지에 아래 항목을 표시한다.

- 데이터 신뢰도 A/B/C
- 가격 확인일과 판매 조건
- 신고번호와 제조사
- 영양 라벨 출처
- 점수 산식 버전
- 결측 또는 제외 이유
- 제휴 여부

## 의료·광고 표현

- “치료”, “예방”, “완치”, “의약품 대체” 표현을 금지한다.
- “피로를 없앤다” 대신 공식 기능성 원문과 일반 정보만 인용한다.
- 1위는 가격효율 산식상 1위이며 개인에게 최선이라고 표현하지 않는다.
- 임신·수유·질환·복약 중 사용자는 전문가 상담 안내를 우선한다.
- 사용자 후기와 전문가 보증을 보유하지 않은 상태에서 암시하지 않는다.

## 출처 표시

- 출처명, 문서/제품명, 근거 필드, URL, 확인일을 함께 표시한다.
- 공공데이터와 제조사 라벨이 다르면 자동 선택하지 않고 검토 상태로 둔다.
- 라벨 이미지는 권리 확인 없이 복제하지 않고 구조화 값과 원문 링크만 사용한다.
- 원문 접근 실패 시 마지막 검증일과 hash를 유지하되 신규 점수 갱신은 중단한다.

## 제휴·광고

- 제휴 여부는 외부 링크 바로 옆에 표시한다.
- 수수료 유무가 순위에 영향을 주지 않는다는 문구를 방법론에 명시한다.
- 광고 카드는 자연 순위 목록 안에 삽입하지 않는다.
- AdSense는 승인 이후 별도 로더 작업으로 진행한다.

## 콘텐츠 품질 Gate

- 출처 없는 숫자 0개
- 기준일 없는 가격 0개
- 보이지 않는 SEO 전용 본문 0개
- visible H1 1개
- 제품 상세마다 고유한 비교·근거 본문 제공
- 방법론에서 산식, 결측, 가격, 신뢰도 규칙을 모두 공개

## 측정

- `page_view`: query 없는 route
- `ranking_view`: 카테고리와 산식 버전
- `filter_apply`: 필터 이름만 전송하고 예산 원값은 제외
- `compare_view`: 제품 수만 전송
- `source_open`: source type
- `affiliate_click`: product ID, seller, affiliate 여부
- 원시 건강정보와 개인 식별정보는 전송하지 않는다.

## 출시 후 검증

- 배포 직후 sitemap, canonical, 404, route별 HTML hash 확인
- GSC는 도메인 속성 또는 `https://shakilabs.com/` URL-prefix 속성을 사용한다.
- sitemap은 리다이렉트 없는 `https://shakilabs.com/nutri/sitemap.xml`을 제출한다.
- GSC 3일·7일·14일 색인 및 선택 canonical 확인
- 30일 후 검색어를 비교·가격·제품명 의도로 분류
- 고노출 저CTR은 title을, 높은 이탈은 근거·비교 UX를 우선 개선
