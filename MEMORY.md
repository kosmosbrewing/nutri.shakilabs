## 확정 결정

- 1차 출시는 프론트엔드 SSG + 버전 관리되는 로컬 데이터로 만든다.
- 공공 영양성분 전체 원본은 `data/raw/` 로컬 스냅샷으로 저장하고 Git에서 제외한다.
- 원본 재현에 필요한 다운로드 스크립트, 행 수, 기준일, SHA-256 manifest는 Git으로 관리한다.
- 브라우저 번들에는 검증·정규화된 제품 subset만 포함한다.
- 별도 NutriMaxBackend를 복사하지 않는다. API가 필요해지면 공유 백엔드의 `nutri` 앱으로 추가한다.
- 로그인, 약물 입력, 임신·수유·질환 개인화는 MVP에서 제외한다.
- 증상에 따른 제품 추천과 치료·예방 표현을 제공하지 않는다.
- 제품 사진은 권리 확인 전 복제하지 않고 텍스트·도형 중심으로 표현한다.
- 최초 공개 제품은 10개이며, 데이터 완전성 gate를 통과한 제품만 랭킹에 포함한다.
- 서로 다른 영양제 카테고리를 한 점수표에 섞지 않고 카테고리별 산식과 근거를 따로 검증한다.
- `unit-price-v1`은 비타민D 10 ug, 프로바이오틱스 10억 CFU, 비타민C·EPA+DHA·마그네슘·칼슘·코엔자임Q10·실리마린 100 mg, MSM 1,000 mg당 가격을 같은 카테고리 안에서만 오름차순 비교한다.
- `category-value-v1`은 `(현재 같은 카테고리의 최저 단위가격 ÷ 제품 단위가격) × 100`이며 최저가 제품을 100점으로 환산한다. 가격 변경 시 재계산하고 카테고리 간 점수는 비교하지 않는다.
- 단위가격에는 원료 형태, 흡수율, 다른 영양소와 개인 적합성을 반영하지 않으며 저가·고함량을 복용 권장으로 표현하지 않는다.
- 공식 카탈로그의 행 수와 예시는 추천, 현재 판매 보장, 고유 상품 수로 표현하지 않는다.
- 제품별 가격은 필수 배송비 포함, 회원·쿠폰 조건 제외 스냅샷을 사용한다.
- 안전 위험은 임의 감점하지 않고 공식 근거가 있는 경고 배지로 분리한다.
- 광고와 자연 순위는 분리하며 제휴 링크는 수수료 가능성을 명시한다.
- AdSense 로더는 승인 후 별도 작업으로 추가한다.

## 공식 데이터 제약

- 전국건강기능식품영양성분정보는 구조화 영양값을 제공하지만 모든 미량영양소와 모든 기존 제품을 완전하게 보장하지 않는다.
- 건강기능식품 품목제조 신고 API는 제품 식별, 섭취방법, 기능성, 주의사항, 기준규격, 원재료의 정본으로 사용한다.
- 2026-06-02 이후 신규 품목신고부터 영양성분 등록이 강화됐으므로 기존 제품 결측이 존재할 수 있다.
- 네이버 검색 쇼핑 API는 2026-07-31 종료 예정이므로 가격 수집 기반으로 사용하지 않는다.
- DUR은 의약품 중심 체계이므로 영양제-약물 상호작용 자동 판정에 직접 사용하지 않는다.
- 식품안전나라 `I0030/C003` 키는 승인됐으며 수집기는 profile의 `KR_FOOD_DAT`만 읽는다. 2026-07-11 현재 공식 sample도 `ERROR-500`이어서 전체 API 스냅샷은 서버 복구 후 재실행한다.
- 신규 18개 제품은 식품안전나라 공개 상세의 신고번호·기준규격·섭취방법을 로컬 evidence로 저장해 핵심 함량과 1일 섭취 개수를 검증한다.

## 운영 규칙

- 데이터 변경은 원문 URL, 확인일, 원문 hash, 검증자 메모를 남긴다.
- `unknown` 영양값을 0으로 바꾸지 않는다.
- 가격이 30일보다 오래되면 랭킹에서 제외한다.
- 단위가격 오퍼 27개의 최근 확인일은 2026-07-11이며 품절·회원가·쿠폰가는 제외한다.
- 점수 산식 버전을 데이터와 함께 노출한다.
- Launch queue는 완료됐으며 후속 작업은 `ISSUES.md` deferred 조건과 `docs/RALPH_LOOP_PLAN.md` gate에 따라 한 이슈씩 진행한다.
